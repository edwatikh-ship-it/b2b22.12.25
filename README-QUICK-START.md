# Быстрый старт проекта

## Документация

Основная документация находится в папке `Docs/`:
- **`Docs/DOCS-INDEX.md`** - индекс всей документации проекта
- **`Docs/PROJECT-DOC.md`** - описание продукта и архитектуры
- **`Docs/PROJECT-RULES.md`** - правила разработки и процесс
- **`Docs/HANDOFF.md`** - журнал успешных изменений
- **`Docs/INCIDENTS.md`** - журнал инцидентов и решений

## Скрипты запуска

### Обычный запуск (окна на весь экран)
```powershell
.\start-dev.ps1
```
Запускает все сервисы в Windows Terminal с окнами нормального размера.

### Тихий запуск (минимизированные окна)
```powershell
.\start-dev-quiet.ps1
```
Запускает все сервисы в минимизированных окнах:
- Windows Terminal - минимизирован
- Chrome - минимизирован (размер окна 800x600)
- Браузер с фронтендом - открывается в фоне

**Примечание:** При обнаружении капчи Chrome автоматически разворачивается на весь экран для удобства решения.

## Сервисы

После запуска доступны:
- **Frontend**: http://localhost:3000/dashboard
- **Backend API**: http://localhost:8001/docs
- **Parser Service**: http://127.0.0.1:9003
- **Chrome CDP**: http://127.0.0.1:9222

## Поведение браузера при парсинге

- **По умолчанию**: Браузер запускается в маленьком окне (800x600)
- **При капче**: Браузер автоматически разворачивается на весь экран
- **После капчи**: Браузер снова минимизируется и продолжает работу

Это позволяет парсеру работать незаметно в фоне, но при необходимости капчи окно автоматически появляется для решения.
