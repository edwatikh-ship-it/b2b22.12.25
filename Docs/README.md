# Документация проекта

## Структура документации

### Основные документы (SSoT)

1. **`DOCS-INDEX.md`** - Индекс всей документации, объясняет структуру и назначение каждого файла
2. **`api-contracts.yaml`** - API контракты (OpenAPI 3.1) - **SSoT для всех API endpoints**
3. **`PROJECT-RULES.md`** - Правила разработки, процесс, safety guards, tooling
4. **`PROJECT-DOC.md`** - Документация продукта, архитектура, решения (ADR), сценарии
5. **`HANDOFF.md`** - Журнал успешных изменений (append-only)
6. **`INCIDENTS.md`** - Журнал инцидентов и решений (append-only)
7. **`SPRINTS.md`** - План спринтов, цели, чеклисты

### Архив

- **`archive/`** - Устаревшие или временные документы

## Как работать с документацией

### При добавлении новой функциональности:
1. Обнови `api-contracts.yaml` если меняется API
2. Обнови `PROJECT-DOC.md` если меняется архитектура или принимаются решения
3. Добавь запись в `HANDOFF.md` после успешной реализации

### При возникновении проблем:
1. Добавь запись в `INCIDENTS.md` с описанием проблемы и решения
2. Обнови `TROUBLESHOOTING.md` в корне проекта если это частая проблема

### При планировании:
1. Обнови `SPRINTS.md` с новыми целями и датами

## Правила

- **SSoT (Single Source of Truth)**: Каждый тип информации имеет один источник правды
- **Real-time**: Документация обновляется сразу после изменений
- **Doc-gate DoD**: "Done" требует обновления документации
- **No duplicates**: Не дублируй информацию в разных файлах
